## New Footguns

**FG-40 — this Reference in state: Field of Trigger-Based Template Sensor** `hard_fail`
`this` is only valid at trigger time. The `state:` field of a trigger-based template sensor is validated at config load — before the entity exists in the state machine. Any `this.*` reference in `state:` causes HA to attempt async resolution against a non-existent entity, raising `asyncio.exceptions.InvalidStateError`. Blast radius: the entire `template:` platform fails, not just the offending sensor. A `this is not none` guard does not help — the asyncio race precedes template evaluation entirely.

The correct pattern is to resolve state data in the `action:` block as a plain Jinja variable, then reference that variable in `state:`. `this` remains valid in `attributes:` (trigger time only) and `value_template:` condition blocks.
